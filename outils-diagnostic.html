Outil de diagnostic Windows

Notre script PowerShell analyse votre poste en profondeur et génère un rapport texte complet — matériel, sécurité, performance, erreurs récentes. Vous gardez le contrôle : aucune donnée n'est transmise sans que vous l'envoyiez vous-même.

✅ 100 % local, 100 % transparent. Le script s'exécute uniquement sur votre poste. Il est en lecture seule (il ne modifie rien) et n'envoie aucune information sur Internet. Le rapport est enregistré sur votre Bureau — c'est vous qui décidez de nous l'envoyer ou non.

// contenu du rapport

Ce que le rapport contient

Tout ce qu'un technicien aurait besoin de vous demander au téléphone — collecté en 30 secondes.

  • Édition, version et build de Windows
  • Modèle du PC, fabricant, numéro de série
  • Processeur, RAM (totale, utilisée, libre)
  • Barrettes mémoire physiques (modèle, vitesse)
  • Disques physiques + état SMART
  • Espace libre par partition
  • État de Windows Defender et signatures
  • Antivirus tiers détectés
  • Pare-feu Windows (3 profils)
  • BitLocker (chiffrement par disque)
  • 15 dernières mises à jour Windows installées
  • Top 10 processus par mémoire et par CPU
  • Programmes lancés au démarrage
  • Services configurés mais arrêtés
  • Erreurs critiques des 7 derniers jours
  • Cartes réseau actives, IP, DNS, passerelle
  • Test de connectivité Internet + DNS
  • Périphériques avec erreur de driver
  • Liste complète des programmes installés
  • Uptime et date du dernier démarrage
// comment l'utiliser

Comment l'utiliser

Choisissez la méthode qui vous convient. Aucune installation requise.

// recommandé

Méthode 1 — Téléchargement

Le plus simple. Vous téléchargez le script, vous le lancez, le rapport apparaît sur votre Bureau.

⬇ Télécharger Diagnostic-BG.ps1

Fichier texte de ~15 Ko. Ouvrez-le dans Bloc-notes avant de l'exécuter si vous voulez vérifier son contenu — il est entièrement lisible.

// powershell

Méthode 2 — Ligne de commande

Pour les utilisateurs à l'aise avec PowerShell. Ouvrez PowerShell (touche Windows, tapez « powershell »), puis copiez-collez :

iex (iwr -UseBasicParsing 'https://bginformatique.ca/outils/Diagnostic-BG.ps1').Content

Cette commande télécharge et exécute le script en mémoire, sans l'enregistrer sur votre disque.

Vérification d'intégrité (recommandé) : Avant d'utiliser la méthode 2, vous pouvez vérifier que le fichier n'a pas été altéré en comparant son empreinte SHA-256. Téléchargez d'abord le fichier, puis dans PowerShell :
(Get-FileHash "$env:USERPROFILE\Downloads\Diagnostic-BG.ps1" -Algorithm SHA256).Hash
La valeur attendue est : 10faf34969ae714d3d6ba981445940c928c5d323ac5c2c3bc29221de6b2633b0

// étape par étape

Étape par étape (Méthode 1)

  1. Téléchargez le fichier

    Cliquez sur le bouton « Télécharger Diagnostic-BG.ps1 » ci-dessus. Le fichier se retrouve dans votre dossier Téléchargements.

  2. Faites un clic droit sur le fichier

    Dans le menu, choisissez « Exécuter avec PowerShell ». Une fenêtre bleue s'ouvre et lance le diagnostic.

    Si Windows demande une confirmation de sécurité (script non signé), répondez O (Oui) puis Entrée. Vous pouvez aussi ouvrir le fichier avec Bloc-notes au préalable pour en vérifier le contenu : tout est en clair.

  3. Attendez 30 à 60 secondes

    Le script affiche sa progression à l'écran (12 étapes). Quand il a terminé, il ouvre automatiquement le rapport dans Bloc-notes.

  4. Vérifiez le rapport

    Le fichier Diagnostic-BG_AAAA-MM-JJ_HH-MM-SS.txt se trouve sur votre Bureau. Lisez-le : si quelque chose ne vous convient pas, supprimez-le, c'est tout.

  5. Envoyez-nous le rapport

    Joignez le fichier .txt à un courriel adressé à information@bginformatique.ca avec un mot sur votre problème. Nous revenons vers vous rapidement.

Si Windows bloque l'exécution avec « cannot be loaded because running scripts is disabled », ouvrez PowerShell en tant qu'administrateur, lancez Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass, puis exécutez le script. Cette commande est temporaire (pour cette fenêtre seulement) et ne modifie pas vos paramètres permanents.

// faq

Questions fréquentes

Est-ce vraiment sans risque ?

Oui. Le script est en lecture seule : il ne modifie aucun paramètre, ne crée ni ne supprime aucun fichier (sauf le rapport sur votre Bureau). Tout son code est lisible en clair dans le fichier .ps1.

Mes données partent-elles sur Internet ?

Non. Le script teste juste votre connexion (un ping vers 8.8.8.8). Aucune donnée du rapport ne quitte votre poste. C'est vous qui décidez de nous envoyer le fichier par courriel.

Faut-il être administrateur ?

Non, mais c'est recommandé. Sans privilèges admin, certaines sections (BitLocker, Defender détaillé, journal d'évènements) seront partielles ou indisponibles. Le rapport indiquera ces limites.

Et sur Mac ou Linux ?

Cet outil est spécifique à Windows. Si vous êtes sur Mac ou Linux, contactez-nous directement par téléphone — nous avons d'autres méthodes de diagnostic à distance.
